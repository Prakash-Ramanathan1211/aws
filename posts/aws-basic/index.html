Aws Points

AWS points

  1. AWS - Amazon Web Services
  2. AWS is a subsidiary of Amazon providing on-demand cloud computing platforms and APIs to individuals, companies, and governments, on a metered pay-as-you-go basis.
  3. These cloud computing web services provide a variety of basic abstract technical infrastructure and distributed computing building blocks and tools.
  4. The AWS technology is implemented at server farms throughout the world, and maintained by the Amazon subsidiary.
  5. Fees are based on a combination of usage (known as a “Pay-as-you-go” model), hardware, operating system, software, or networking features chosen by the subscriber required availability, redundancy, security, and service options.
  6. Subscribers can pay for a single virtual AWS computer, a dedicated physical computer, or clusters of either.
  7. AWS is separated into different services; each can be configured in different ways based on the user’s needs.
  8. It provides a mixture of infrastructure as a service (IaaS), platform as a service (PaaS) and packaged software as a service (SaaS) offerings.
  9. AWS services can offer an organization tools such as compute power, database storage and content delivery service
  10. More than 100 services comprise the Amazon Web Services portfolio, including those for compute, databases, infrastructure management, application development and security

Availability:

  1. Amazon Web Services provides services from dozens of data centers spread across availability zones (AZs) in regions across the world.
  2. An AZ is a location that contains multiple physical data centers.
  3. A region is a collection of AZs in geographic proximity connected by low-latency network links.
  4. A business will choose one or multiple availability zones for a variety of reasons, such as compliance and proximity to end customer.
  5. An AWS customer can spin up virtual machines (VMs) and replicate data in different AZs to achieve a highly reliable infrastructure that is resistant to failures of individual servers or an entire data center.
  6. Amazon Elastic Compute Cloud (EC2) is a service that provides virtual servers – called EC2 instances – for compute capacity.
  7. The EC2 service offers dozens of instance types with varying capacities and sizes, tailored to specific workload types and applications, such as memory-intensive and accelerated-computing jobs.
  8. AWS also provides an Auto Scaling tool to dynamically scale capacity to maintain instance health and performance.

Storage:

  1. Amazon Simple Storage Service (S3) provides scalable object storage for data backup, collection and analytics.
  2. Amazon Elastic Block Store provides block-level storage volumes for persistent data storage when using EC2 instances.
  3. Amazon Elastic File System offers managed cloud-based file storage.
  4. A business can also migrate data to the cloud via storage transport devices, such as AWS Snowball and Snowmobile.
  5. It use AWS Storage Gateway to enable on-premises apps to access cloud data.

Databases, data management:

  1. The Amazon Relational Database Service – which includes options for Oracle, SQL Server, PostgreSQL, MySQL, MariaDB and a proprietary high-performance database called Amazon Aurora – provides a relational database management system for AWS users.
  2. AWS also offers managed NoSQL databases through Amazon DynamoDB.
  3. An AWS customer can use Amazon ElastiCache and DynamoDB Accelerator as in-memory and real-time data caches for applications.
  4. Amazon Redshift offers a data warehouse, which makes it easier for data analysts to perform business intelligence (BI) tasks.

Migration, hybrid cloud:

  1. AWS includes various tools and services designed to help users migrate applications, databases, servers and data onto its public cloud.
  2. The AWS Migration Hub provides a location to monitor and manage migrations from on premises to the cloud.
  3. Once in the cloud, EC2 Systems Manager helps an IT team configure on-premises servers and AWS instances.
  4. Amazon also has partnerships with several technology vendors that ease hybrid cloud deployments.
  5. VMware Cloud on AWS brings software-defined data center technology from VMware to the AWS cloud.
  6. Red Hat Enterprise Linux for Amazon EC2 is the product of another partnership, extending Red Hat’s operating system to the AWS cloud.

Networking:

  1. An Amazon Virtual Private Cloud (Amazon VPC) gives an administrator control over a virtual network to use an isolated section of the AWS cloud.
  2. AWS automatically provisions new resources within a VPC for extra protection.
  3. Admins can balance network traffic with the Elastic Load Balancing (ELB) service, which includes the Application Load Balancer and Network Load Balancer.
  4. AWS also provides a domain name system called Amazon Route 53 that routes end users to applications.
  5. An IT professional can establish a dedicated connection from an on-premises data center to the AWS cloud via AWS Direct Connect.

Developer tools:

  1. A developer can take advantage of AWS command-line tools and software development kits (SDKs) to deploy and manage applications and services.
  2. The AWS Command Line Interface, which is Amazon’s proprietary code interface.
  3. A developer can use AWS Tools for Powershell to manage cloud services from Windows environments.
  4. Developers can use AWS Serverless Application Model to simulate an AWS environment to test Lambda functions.
  5. Amazon API Gateway enables a development team to create, manage and monitor custom application program interfaces (APIs) that let applications access data or functionality from back-end services.
  6. API Gateway manages thousands of concurrent API calls at once.
  7. AWS also provides a packaged media transcoding service – Amazon Elastic Transcoder – and a service that visualizes workflows for microservices-based applications – AWS Step Functions.
  8. A developer can also store code in Git repositories with AWS CodeCommit and evaluate the performance of microservices-based applications with AWS X-Ray.
  9. A development team can also create continuous integration and continuous delivery pipelines with services like AWS CodePipeline.
  10. A development team can also create continuous integration and continuous delivery pipelines with services like AWS CodeBuild.
  11. A development team can also create continuous integration and continuous delivery pipelines with services like AWS CodeDeploy.
  12. A development team can also create continuous integration and continuous delivery pipelines with services like AWS CodeStar.

Management and monitoring:

  1. An admin can manage and track cloud resource configuration via AWS Config and AWS Config Rules.
  2. Those tools, along with AWS Trusted Advisor, can help an IT team avoid improperly configured and needlessly expensive cloud resource deployments.
  3. WS provides several automation tools in its portfolio.
  4. An admin can automate infrastructure provisioning via AWS CloudFormation templates, and also use AWS OpsWorks and Chef to automate infrastructure and system configurations.
  5. An AWS customer can monitor resource and application health with Amazon CloudWatch.
  6. The AWS Personal Health Dashboard, as well as use AWS CloudTrail to retain user activity and API calls for auditing.

Security and governance:

  1. AWS provides a range of services for cloud security, including AWS Identity and Access Management, which allows admins to define and manage user access to resources.
  2. An admin can also create a user directory with Amazon Cloud Directory, or connect cloud resources to an existing Microsoft Active Directory with the AWS Directory Service.
  3. The AWS Organizations service enables a business to establish and manage policies for multiple AWS accounts.
  4. Amazon Web Services has also introduced tools that automatically assess potential security risks.
  5. Amazon Inspector analyzes an AWS environment for vulnerabilities that might impact security and compliance.
  6. Amazon Macie uses machine learning (ML) technology to protect sensitive cloud data.
  7. AWS also includes tools and services that provide software- and hardware-based encryption, protect against DDoS attacks, provision Secure Sockets Layer (SSL) and Transport Layer Security (TLS) certificates and filter potentially harmful traffic to web applications.
  8. The AWS Management Console is a browser-based graphical user interface (GUI) for AWS.
  9. The Management Console can be used to manage resources in cloud computing, cloud storage and security credentials.
  10. The AWS Console interfaces with all AWS resources.

Big data management and analytics:

  1. AWS includes a variety of big data analytics and application services
  2. Amazon Elastic MapReduce, which offers a Hadoop framework to process large amounts of data.
  3. Amazon Kinesis, which provides several tools to process and analyze streaming data.
  4. AWS Glue, which is a service that handles extract, transform and load jobs.
  5. Amazon Elasticsearch Serviceenables a team to perform application monitoring, log analysis and other tasks with the open source Elasticsearch tool.
  6. Amazon Athena for S3, which allows analysts to query data.
  7. Amazon QuickSight, which help analysts visualize data.
